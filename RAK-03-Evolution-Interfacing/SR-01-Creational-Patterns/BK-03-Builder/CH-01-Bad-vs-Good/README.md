# CH-01: Bad vs Good (Builder)

> "Meninggalkan Constructor 'Raksasa' demi Kelenturan Rakitan."

## 1. Skenario Kekacauan (The Problem)
Kita akan melihat bagaimana sebuah kelas `Rumah` yang punya banyak parameter opsional membuat pusing programmer yang menggunakannya. Kode tersebut sangat kaku dan sulit dibaca.

## 2. Analogy
Ini seperti membandingkan **Membeli Makanan Paket Hemat** (Kaku, tidak bisa dirubah isinya) dengan **Prasmanan** (Anda bebas ambil apa saja sesuai keinginan).

# BK-01: GoF Three Pillars

> "Tiga pilar utama penyangga arsitektur perangkat lunak."

## 1. Skenario Kekacauan (The Problem)
Jika kita menganggap semua masalah kode itu sama, kita akan menggunakan alat yang salah. Misalnya, menggunakan pola komples hanya untuk membuat objek sederhana, atau menggunakan pola pembuatan saat masalahnya adalah komunikasi antar objek.

## 2. Analogy
Bayangkan membangun sebuah **Restoran**.
- **Creational**: Memilih bahan baku dan membeli peralatan masak (Proses Pengadaan).
- **Structural**: Merancang tata letak meja, dapur, dan kasir (Proses Desain Ruang).
- **Behavioral**: Menentukan bagaimana pelayan mencatat pesanan dan memberikannya ke koki (Proses Komunikasi).

## 3. Everyday Deep Dive
Dalam buku ini, kita akan melihat pembagian 23 pola legendaris:
- **5 Creational Patterns**: Singleton, Factory, etc.
- **7 Structural Patterns**: Adapter, Facade, etc.
- **11 Behavioral Patterns**: Observer, Strategy, etc.

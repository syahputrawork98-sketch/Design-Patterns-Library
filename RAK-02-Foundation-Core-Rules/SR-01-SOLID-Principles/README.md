# SR-01: SOLID Principles

> "Lima pilar penyangga kode yang bersih dan adaptif."

## 1. Skenario Kekacauan (The Problem)
Pernahkah Anda merombak database dan tiba-tiba seluruh fitur di aplikasi Anda rusak? Itu terjadi karena kode Anda "tahu terlalu banyak" detail. SOLID memandu kita untuk membangun sistem yang terklasifikasi dengan rapi.

## 2. Analogy
SOLID seperti **Kotak Perkakas Modern**. Jika palu Anda patah, Anda tidak perlu mengganti seluruh kotak—cukup ganti palunya saja. Setiap alat memiliki tempat dan fungsinya sendiri.

## 3. Everyday Deep Dive
Sub-Rak ini membagi Prinsip SOLID menjadi 5 Buku utama:
- **BK-01**: Single Responsibility (Fokus Satu Tugas).
- **BK-02**: Open/Closed (Siap Tambah, Anti Modif).
- **BK-03**: Liskov Substitution (Anak yang Taat Aturan).
- **BK-04**: Interface Segregation (Jangan Paksa Klien).
- **BK-05**: Dependency Inversion (Berteman dengan Abstraksi).

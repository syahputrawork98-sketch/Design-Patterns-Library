# SR-01: Tooling Philosophy

> "Bekerja Lebih Cerdas, Bukan Lebih Keras."

## 1. Skenario Kekacauan (The Problem)
Software engineering modern sudah terlalu kompleks untuk dikerjakan secara manual. Tanpa filosofi alat yang benar, kita akan sibuk mengurusi konfigurasi daripada menulis fitur.

## 2. Analogy
Bayangkan **Chef Profesional**. Mereka tidak memotong bawang secara manual untuk ribuan tamu. Mereka punya *Food Processor* yang bekerja dalam hitungan detik dengan hasil yang seragam. Itulah filosofi Tooling.

## 3. Everyday Deep Dive
Filosofi alat di Rak ini berfokus pada 3 pilar:
1. **Automation**: Biarkan mesin melakukan pekerjaan berulang.
2. **Standardization**: Pastikan semua orang di tim menulis dengan gaya yang sama.
3. **Prevention**: Temukan bug *sebelum* user yang menemukannya.
